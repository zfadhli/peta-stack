# Advanced Migration Usage

## Migration Structure

A migration file exports `up` and `down` functions:

```ts
import type { Kysely } from "kysely"

export async function up(db: Kysely<unknown>): Promise<void> {
  await db.schema
    .createTable("users")
    .addColumn("id", "integer", (c) => c.autoIncrement().primaryKey())
    .addColumn("name", "varchar(255)", (c) => c.notNull())
    .addColumn("email", "varchar(255)", (c) => c.notNull().unique())
    .execute()
}

export async function down(db: Kysely<unknown>): Promise<void> {
  await db.schema.dropTable("users").execute()
}
```

### Naming Convention

Migration file names determine execution order. Convention: `001_description.ts`, `002_description.ts`, etc.

## Rollback Strategies

### Batch Rollback

`runner.down()` rolls back the **last batch** of migrations. A batch is all migrations applied in a single `runner.up()` call.

```ts
const runner = createMigrationRunner(kysely)

// Apply 3 migrations (batch 1)
await runner.up([migration1, migration2, migration3])

// Apply 2 more (batch 2)
await runner.up([migration4, migration5])

// Rolls back migration5 and migration4 (batch 2, in reverse order)
await runner.down()
```

### Individual Rollback

To roll back a specific number of steps, call `down()` multiple times:

```ts
// Roll back 2 batches
await runner.down()
await runner.down()
```

## Data Migrations

Migrations can transform existing data, not just schema:

```ts
export async function up(db: Kysely<unknown>): Promise<void> {
  // Add new column
  await db.schema
    .alterTable("users")
    .addColumn("display_name", "varchar(255)")
    .execute()

  // Backfill data
  await db.updateTable("users")
    .set({ display_name: db.sql`COALESCE(NULLIF(name, ''), email)` })
    .execute()
}

export async function down(db: Kysely<unknown>): Promise<void> {
  await db.schema
    .alterTable("users")
    .dropColumn("display_name")
    .execute()
}
```

## Migration Generator

The generator creates migration code from model definitions:

```ts
import { createMigrationGenerator } from "peta-migrate"

const gen = createMigrationGenerator()

// Generate an initial migration from all registered models
const code = gen.generateInitialMigration(models)
// Returns a string containing the migration file content
```

The generator reads column types, constraints, and relations from the model definitions and produces Kysely schema builder code.

## Seed Data

peta-migrate does not include a built-in seed system. Use the ORM directly for seeding:

```ts
import { User } from "./src/db/schema.js"

async function seed() {
  await User.insertMany([
    { name: "Alice", email: "alice@example.com" },
    { name: "Bob", email: "bob@example.com" },
  ])
  console.log("Database seeded!")
}

await seed()
```

## Multi-Database Considerations

peta-migrate uses Kysely, which abstracts database-specific SQL. However, some migration patterns differ:

| Feature | SQLite | PostgreSQL | MySQL |
|---------|--------|------------|-------|
| Auto-increment | `AUTOINCREMENT` | `GENERATED BY DEFAULT AS IDENTITY` | `AUTO_INCREMENT` |
| Column alterations | Limited — often requires recreate | Full `ALTER TABLE` support | Full `ALTER TABLE` support |
| Transactions | DDL wrapped in transactions | DDL wrapped in transactions | DDL auto-commits |

For production, test migrations against each target database before deploying.

## Troubleshooting

### Migration Already Applied

If a migration fails mid-way, the tracking table already has it marked as complete:

```ts
// Check status
const status = await runner.status()
console.log("Completed:", status.completed)
console.log("Pending:", status.pending)

// Manually mark a failed migration as not completed
await runner.kysely
  .deleteFrom("_migrations")
  .where("name", "=", "002_broken_migration")
  .execute()
```

### Migration Order Conflicts

If two branches both add `003_*.ts`, rename the later one before merging.

### Cannot Drop Column (SQLite)

SQLite doesn't support `DROP COLUMN`. Use a recreate pattern:

```ts
export async function up(db: Kysely<unknown>): Promise<void> {
  await db.schema
    .createTable("users_v2")
    .addColumn("id", "integer", (c) => c.autoIncrement().primaryKey())
    .addColumn("name", "varchar(255)")
    .execute()

  await db.insertInto("users_v2").execute(
    db.selectFrom("users").select(["id", "name"])
  )

  await db.schema.dropTable("users").execute()
  await db.schema.alterTable("users_v2").renameTo("users").execute()
}
```
